All classes are defined in static/css/custom.css. Use these instead of hardcoding styles inline.

02

Buttons

6 button variants. Use Tiaki classes — not Bootstrap's btn-primary. Buttons in page headers use the sm variants.

When to use each
ClassUse forWhere
btn-pfPrimary actionHero, forms
btn-pf-outlineSecondary action (on dark bg)Hero, navbar
btn-pf-fullFull-width submitAuth forms
btn-sm-pfPrimary small actionPage headers
btn-sm-outlineSecondary small actionPage headers, tables
btn-sm-dangerDestructive actionTables, modals
btn-sm-amberWarning action (deactivate etc)User management
HTML
<!-- Primary --> <a href="..." class="btn-pf">Label</a> <!-- Small primary (page header) --> <a href="..." class="btn-sm-pf"> <i class="bi bi-plus-lg me-1"></i>New Item </a> <!-- Danger in a table row --> <button class="btn-sm-danger">Delete</button>

05

Tables

Use pf-table inside a panel. Always wrap in style="overflow-x:auto" for mobile. Always include an empty state row.

Recent catches
Last 5 records
Date Line Trap Species Status
19 Mar 2026 North Line NL-03 Stoat Sprung
18 Mar 2026 Lake Route LR-01 Possum Still set
HTML
<div class="panel"> <div class="panel-header">...</div> <div style="overflow-x:auto;"> <table class="pf-table"> <thead> <tr><th>Column</th></tr> </thead> <tbody> {% for row in rows %} <tr> <td>{{ row.value }}</td> </tr> {% else %} <tr> <td colspan="X" style="text-align:center; padding:30px; color:#6b7c72;"> No records found. </td> </tr> {% endfor %} </tbody> </table> </div> </div>
06

Badges

Specific badge classes for species, trap status, account status, and user roles. Don't use Bootstrap's generic .badge for these.

Species
Stoat Possum Ship Rat
Trap status
Sprung Still set Removed
Trap physical status
Active Retired
Account status
Active Inactive
User roles
Admin Operator Observer
HTML — conditional badge patterns
<!-- Species --> <span class="badge-species">{{ record.species_caught }}</span> <!-- Trap status (use conditional) --> {% if 'Sprung' in record.status %} <span class="badge-status-sprung">{{ record.status }}</span> {% elif record.status == 'Removed' %} <span class="badge-status-removed">{{ record.status }}</span> {% else %} <span class="badge-status-set">{{ record.status }}</span> {% endif %} <!-- Trap physical status --> {% if trap.is_retired %} <span class="trap-status-badge trap-status-retired">Retired</span> {% else %} <span class="trap-status-badge trap-status-active">Active</span> {% endif %} <!-- User role --> {% if user.role == 'Admin' %} <span class="badge-role-admin">Admin</span> {% elif user.role == 'Operator' %} <span class="badge-role-operator">Operator</span> {% else %} <span class="badge-role-observer">Observer</span> {% endif %} <!-- Account status --> {% if user.account_status == 'active' %} <span class="badge-active">Active</span> {% else %} <span class="badge-inactive">Inactive</span> {% endif %}
07

Notices

Inline info/warning banners. Use notice with a colour modifier. Note: red is not defined — use Bootstrap's alert-danger for errors inside forms.

Green — success or read-only info (Observer notice).
Blue — informational message, login redirect notice.
Amber — warning, e.g. not assigned to any lines yet.
HTML
<!-- colour options: green · blue · amber --> <div class="notice blue"> <i class="bi bi-info-circle"></i> <span>Message here.</span> </div>

09

Forms

Wrap form sections inside a panel. Use field-group per field, form-hint for help text, required-star for required indicators.

Personal details
Account
Assign the appropriate role for this user.
Cancel

* Required fields

HTML
<div class="panel"> <div class="panel-header"> <div class="panel-title">Section</div> </div> <div class="panel-body"> <div class="form-section-label">Sub-section</div> <form method="POST"> <div class="row g-3"> <div class="col-md-6 field-group"> <label class="form-label"> Label <span class="required-star">*</span> </label> <input type="text" class="form-control" required> <div class="form-hint">Help text.</div> </div> </div> <div class="d-flex gap-2 mt-3"> <button type="submit" class="btn-pf">Save</button> <a href="..." class="btn-pf-outline">Cancel</a> </div> </form> </div> </div>

11

Jinja global variables

Available in every template automatically via the context processor in __init__.py. Never pass these from routes — they're already there.

VariableReturnsUse for
{{ site_name }} Tiaki App name in text, titles
{{ site_tagline }} Predator Trapping & Monitoring Subtitle text
{{ logo_url }} static/images/logo.png path Full logo <img> src
{{ icon_url }} static/images/icon.png path Icon, favicon <img> src
{{ nav_first_name }} Logged-in user's first name Navbar display name
{{ nav_full_name }} Logged-in user's full name Sidebar display name
{{ nav_profile_photo }} Photo filename or None Navbar avatar image
Usage examples
<!-- Site name in text --> <p>Welcome to {{ site_name }}</p> <!-- Logo image --> <img src="{{ logo_url }}" alt="{{ site_name }}" style="height:36px;"> <!-- Icon image --> <img src="{{ icon_url }}" alt="{{ site_name }}" style="height:24px;"> <!-- Page title --> <title>{% block title %}Page{% endblock %} | {{ site_name }}</title> <!-- Greeting --> <p>Hello, {{ nav_first_name }}!</p>
