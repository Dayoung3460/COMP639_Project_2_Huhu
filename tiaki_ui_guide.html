HTML — conditional badge patterns
<!-- Species --> <span class="badge-species">{{ record.species_caught }}</span> <!-- Trap status (use conditional) --> {% if 'Sprung' in record.status %} <span class="badge-status-sprung">{{ record.status }}</span> {% elif record.status == 'Removed' %} <span class="badge-status-removed">{{ record.status }}</span> {% else %} <span class="badge-status-set">{{ record.status }}</span> {% endif %} <!-- Trap physical status --> {% if trap.is_retired %} <span class="trap-status-badge trap-status-retired">Retired</span> {% else %} <span class="trap-status-badge trap-status-active">Active</span> {% endif %} <!-- User role --> {% if user.role == 'Admin' %} <span class="badge-role-admin">Admin</span> {% elif user.role == 'Operator' %} <span class="badge-role-operator">Operator</span> {% else %} <span class="badge-role-observer">Observer</span> {% endif %} <!-- Account status --> {% if user.account_status == 'active' %} <span class="badge-active">Active</span> {% else %} <span class="badge-inactive">Inactive</span> {% endif %}
07

Notices

Inline info/warning banners. Use notice with a colour modifier. Note: red is not defined — use Bootstrap's alert-danger for errors inside forms.

Green — success or read-only info (Observer notice).
Blue — informational message, login redirect notice.
Amber — warning, e.g. not assigned to any lines yet.
HTML
<!-- colour options: green · blue · amber --> <div class="notice blue"> <i class="bi bi-info-circle"></i> <span>Message here.</span> </div>

15

Modals

Confirmation modals use Bootstrap Modal with pop-up-modal for correct z-index layering above the sidebar. Use btn-sm-outline for Cancel and btn-sm-danger for destructive confirms.

Retire trap

Are you sure you want to retire trap NL-03?

This trap will be marked as Retired.

Key rules
RuleDetails
Z-indexAdd pop-up-modal to the modal so it appears above the sidebar (z-index 2001)
Cancel buttonUse btn-sm-outline with data-bs-dismiss="modal"
Confirm buttonUse btn-sm-danger for destructive actions, btn-sm-pf for safe actions
Footer layoutRight-align buttons with d-flex justify-content-end gap-2
HTML
<div class="modal fade pop-up-modal" id="retire-trap-modal" tabindex="-1"> <div class="modal-dialog"> <div class="modal-content"> <div class="modal-header"> <h5 class="modal-title">Retire trap</h5> <button class="btn-close" data-bs-dismiss="modal"></button> </div> <div class="modal-body"> <p>Are you sure you want to retire trap <strong>{{ code }}</strong>?</p> </div> <div class="modal-footer"> <button class="btn-sm-outline" data-bs-dismiss="modal">Cancel</button> <button type="submit" class="btn-sm-danger">Yes, retire</button> </div> </div> </div> </div> <!-- CSS (already in custom.css) --> .modal-backdrop { z-index: 2000; } .modal.pop-up-modal { z-index: 2001; }
16

Responsive breakpoints

Key breakpoints used in custom.css. The sidebar collapses on mobile, page headers stack vertically, and tables get horizontal scroll.

BreakpointWhat changes
< 768pxSidebar hidden, page headers stack vertically, action buttons go full width
< 960pxGroup photo reduced height, tighter spacing
< 576pxSmallest layout — minimal padding, compact cards
> 1200pxAuth card max-width applied, wider stat card grid
Patterns to follow
/* Always wrap tables for mobile scroll */ <div class="pf-table-responsive"> <table class="pf-table">...</table> </div> /* Use d-none/d-md-flex to show/hide elements */ <div class="d-none d-md-flex">Desktop only</div> <select class="d-md-none">Mobile only</select> /* Sidebar collapses via JS toggle + .collapsed class */ <aside class="sidebar collapsed">...</aside>
