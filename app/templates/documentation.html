The Problem

What is Tiaki?

Predator Free Lincoln University (PF-LU) is a fictional volunteer group running a predator-control initiative across the Lincoln University campus. They maintain a network of traps targeting introduced predator species — possums, stoats, rats, and weasels — that threaten New Zealand's native wildlife. Previously, PF-LU relied on paper-based forms and spreadsheets, causing data errors, slow reporting, and difficulty tracking trends over time.

Tiaki (Māori for "to guard" or "to protect") is a custom beta web application that replaces this manual process. It allows field operators to log trap checks from the field, generates visual reports automatically, tracks individual trap status, and aligns data with the trap.nz national CSV import format.

Trap Network Management

Admins create and manage trap lines and individual traps with GPS coordinates across the Lincoln University campus.

Field Data Entry

Operators log trap check results from the field — species caught, trap status, bait type, condition, and strikes.

Reporting & Analytics

Visual charts and filterable tables give all users a clear view of predator activity trends across all trap lines.

CSV Export

Download catch record data as CSV — formatted to be compatible with trap.nz's national data import requirements.

Role-Based Access

Three roles — Observer, Operator, Admin — each with the right level of access for their responsibilities.

Mobile Friendly

Fully responsive design so field operators can log data directly from their phones while checking traps on campus.

Access Control

User Roles

Three roles with role-based permissions enforced via a custom @role_required() decorator on every route.

👁 Observer Volunteer / Visitor
  • ✅ View trap lines and traps on map
  • ✅ Browse and filter catch records
  • ✅ View incidental observations
  • ✅ Download CSV data
  • ✅ View reports and charts
  • ✅ View own profile
  • ❌ Cannot add or edit records
🔧 Operator Field Worker
  • ✅ Everything Observers can do
  • ✅ Add new trap catch records
  • ✅ Edit own catch records
  • ✅ Record incidental observations
  • ✅ View assigned lines with map
  • ✅ View operator dashboard
⚙️ Admin System Administrator
  • ✅ Everything Operators can do
  • ✅ Create & manage trap lines/traps
  • ✅ Assign operators to lines
  • ✅ Retire lines and traps
  • ✅ Manage all users & roles
  • ✅ Manage species, statuses, bait types

Product

Key Features

A walkthrough of the major features built across both sprints.

🗺️
Interactive Trap Maps
Leaflet.js + LINZ base tiles

Every trap line and individual trap is plotted on an interactive map using Leaflet.js with LINZ topographic base tiles. Users can click traps to view details and navigate to the line detail page. Active and retired traps are visually differentiated.

  • Lines index — all trap lines with map overview
  • Line detail — individual trap locations per line
  • My Lines — operator's assigned lines with map
  • Trap retirement preserved on map with visual cue
📊
Reports & Charts
Chart.js with live DB queries

The reports page features four interactive Chart.js charts with filters for line, operator, and date range. All data updates dynamically from the PostgreSQL database.

  • Catch trend over time — weekly line chart
  • Captures by species — doughnut chart with 5% threshold grouping
  • Catches by trap line — horizontal bar chart, top 10 with "Others" modal
  • Trap status distribution — stacked bar chart per line
  • Key insights panel — busiest line, maintenance alerts, capture rate
  • Print/PDF export — charts captured as PNG via canvas
👥
User Management
Admin role — full control

Admins have a full user management interface with search, filtering, sorting, and the ability to view detailed profiles, change roles, and activate/deactivate accounts.

  • Search by name or username
  • Filter by role and account status
  • Sort by name, role, status, date joined, last login
  • Modal confirmation for activate/deactivate
  • Role badge styling on edit role modal
  • Admin cannot deactivate own account
🔐
Authentication
Secure, session-based

Full authentication system with role-based access control enforced via a custom decorator on every route. Password reset via email link.

  • Bcrypt password hashing (12 rounds)
  • Custom @role_required() decorator
  • Back-button protection after logout (cache headers)
  • Account deactivation check on every request
  • Password reset via Gmail SMTP token link
  • Read-only profile view + separate edit profile

Database Design

PostgreSQL with ENUMs for fixed value sets, foreign key constraints, and a normalised schema. Data fields align with the trap.nz CSV import format.

Lookup tables

species, trap_statuses, and bait_types are managed lookup tables with name as the primary key. Foreign key constraints with ON UPDATE CASCADE ensure data consistency if a lookup value is renamed by an Admin.
